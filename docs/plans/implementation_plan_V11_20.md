# [V11.20] 히어로 컴포넌트 물리적 격리 및 클린업 계획

시네마틱 와이프(Wipe) 연출에 최적화된 컴포넌트 구조로 재편하고, 온모드(ON) 진입 시 불필요한 자원(파티클 등)을 완전히 제거하여 성능과 코드 가독성을 극대화합니다.

## 🚦 핵심 변경 사항 (User Review Required)

> [!IMPORTANT]
> **하이브리드 구조에서 완전 격리 구조로의 전환**: 
> 기존에는 하나의 컴포넌트(`HeroSlogan` 등)가 ON/OFF 상태를 모두 들고 내부에서 제어했으나, 이제는 **[OFF 전용 컴포넌트]**와 **[ON 전용 컴포넌트]**로 물리적으로 분리합니다. 이를 통해 레이어 간섭 문제를 원천 차단합니다.

## 🛠️ 주요 수정 내역

### 1. [컴포넌트 분리] Slogan 및 Phrase 구조 재편
- **[NEW] `HeroSloganOff.tsx` / `HeroSloganOn.tsx`**: `HeroSlogan.tsx` 내부의 복잡한 조건부 로직을 제거하고, 각 모드에 최적화된 독립된 컴포넌트로 분산합니다.
- **[REFACTOR] `HeroPhraseLayer.tsx`**: 오직 **온모드(ON)** 상태에서만 렌더링되도록 상위 뷰(`HeroPCView` 등)에서 조건부 마운트 로직을 강화합니다.

### 2. [자원 최적화] 온모드 클린업 (Clean-up)
- **파티클 캔버스 언마운트**: 전환이 완료되어 온모드에 진입하면 `canvas` 컴포넌트를 DOM에서 완전히 제거하여 GPU 점유율을 0으로 만듭니다.
- **램프 이펙트 제거**: 오프모드의 시나리오인 '램프 비추기' 연출을 온모드에서는 코드 레벨에서 아예 배제합니다.

### 3. [뷰 레이아웃 정립] 3단 뷰 로직 간소화
- **`HeroPCView.tsx` / `HeroTabletView.tsx` / `HeroMobileView.tsx`**: 
    - `{ !isOn && <HeroOffComponents /> }`
    - `{ isOn && <HeroOnComponents /> }`
- 위와 같은 명확한 조건부 렌더링 구조로 변경하여, `zIndex`나 `pointer-events` 등의 물리적 충돌 가능성을 0%로 제거합니다.

## ❓ 열린 질문 (Open Questions)

- **와이프 타이밍**: 컴포넌트가 교체될 때 '찰나의 비어 보임'을 방지하기 위해, 와이프가 화면을 100% 가린 시점에 상태를 교체하는 `sequenceStep` 기반의 정밀 트리거를 적용할 예정입니다.

## ✅ 검증 계획

### 자동화 테스트
- `pnpm dev` 실행 시 빌드 오류 및 런타임 오류 확인.
- 브라우저 개발자 도구의 'Performance' 탭에서 온모드 진입 후 파티클 연산(Canvas)이 중단 여부 검증.

### 수동 검증
- 모바일/태블릿에서 전환 시점에 문구들이 매끄럽게 교체되는지 확인.
- 온모드에서 배경의 미니멀함이 순수하게 유지되는지 확인.
