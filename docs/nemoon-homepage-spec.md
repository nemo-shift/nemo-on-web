**배경색 인터폴레이션 (Scroll-driven Color Interpolation)**

섹션 전환 구간에서 배경색이 스크롤 progress에 따라 서서히 전환되는 방식.
네모 오브젝트 모핑과 별개로 배경색 자체를 부드럽게 변화시킴.

구현: GSAP ScrollTrigger `onUpdate` 콜백에서 두 색상을 lerp(선형보간)하여 CSS 변수(`--bg`)에 실시간 주입.

```
scrollProgress 0 → 1
bgColor = lerp(startColor, endColor, progress)
document.documentElement.style.setProperty('--bg', bgColor)
```

## 6. 반응형 정책

**레이아웃** — 뷰포트 너비 기준으로 분기
**인터랙션** — 입력장치 기준으로 분기 (레이아웃과 독립)

**레이아웃 breakpoint**

| breakpoint | 범위             |
| ---------- | ---------------- |
| PC         | `≥ 1024px`       |
| 태블릿     | `768px ~ 1023px` |
| 모바일     | `< 768px`        |

> 태블릿 가로(≥ 1024px)는 PC 레이아웃으로 자연스럽게 처리됨.

| 환경   | 정책                                                               |
| ------ | ------------------------------------------------------------------ |
| PC     | 여유 있는 화이트스페이스, 뷰포트 비례(`vw`, `vh`) 기반 간격.       |
| 태블릿 | 섹션별로 PC 또는 모바일 레이아웃 중 명시. 기본은 PC 레이아웃 따름. |
| 모바일 | 수직형 배치, 터치 고려 간격 확대. 세로 스택형 기본.                |

**인터랙션 분기**

| 조건        | 판단 기준         | 적용                                   |
| ----------- | ----------------- | -------------------------------------- |
| 마우스 있음 | `pointer: fine`   | 커스텀 커서 활성, 호버 인터랙션 활성   |
| 터치만 있음 | `pointer: coarse` | 커스텀 커서 비활성, 터치 인터랙션 활성 |

> 레이아웃과 인터랙션은 독립적으로 작동.
> 태블릿 가로(PC 레이아웃)여도 터치 기기면 커서 없음.
> PC에서 창을 줄여도 마우스 있으면 커서 유지.

> 각 섹션의 환경별 세부 동작은 해당 섹션 "반응형 레이아웃" 항목에 기재.

## 8. 네모 Morphing 전체 시퀀스

네모는 홈페이지 전체를 관통하는 하나의 살아있는 오브젝트.
섹션 전환 시 변형이 시작되고, 다음 섹션에 어느 정도 진입했을 때 완성됨.
스크롤 progress에 연동 — 역방향 되감기도 자연스럽게 동작.

```
히어로
└── 네모 상태: [결] 뒤 작은 네모 (Shared Element 시작점)
    ↓ 스크롤 — 작은 네모가 확장되며 Pain & 공명 전체 배경이 됨
Pain & 공명
└── 히어로 네모: Pain & 공명 배경(`#0D1A1F`)으로 전환 완료 (Shared Element 역할 종료)
    새 네모 등장: `#f0ebe3` 테두리, 오른쪽 위치
    ↓ 테두리 → 크림 채워짐 → 틸로 색 전환 + 세로 긴 박스로 변형
메시지
└── 진입 후 완성: 틸(`#0891b2`) 세로 긴 작은 박스, 중앙
    ↓ 변형 시작
ForWho
└── 진입 후 완성: 버스 사진이 채워진 가로 박스 (위아래 배경 보임)
    ↓ 이후 미정
브랜드 스토리 / CTA
└── 미정
```

참고링크(https://www.easymedia.net/kr/main)

#### 구현 방식

특정 컴포넌트(히어로 섹션 [결] 뒤의 네모)가 섹션 스크롤에 따라 사라지지 않고 'Shared Element'로 유지되면서 변형되는 인터랙션으로 구현.

1. 시작 상태: 히어로 섹션 [결] 뒤 특정 위치에서 특정 크기와 색상으로 시작.
2. 전환 과정 (ScrollTrigger Scrub):
   - 히어로 → Pain & 공명: 작은 네모가 확장되며 섹션 전체 배경(`#0D1A1F`)이 됨. (Shared Element 역할 종료)
   - Pain & 공명: 새 테두리 네모 등장. 안에서 Pain 포인트, 공명 문장 진행.
   - Pain & 공명 → 메시지: 테두리 → 크림 채워짐 → 틸 세로 긴 박스로 변형.
   - 메시지 → ForWho: 세로 긴 박스 → 가로 박스 확장 + 버스 사진 채워짐.
3. 구현 원칙:
   - Pain & 공명 이후 네모는 DOM에서 단일 `div`를 유지할 것.
   - GSAP ScrollTrigger의 `scrub`을 사용하여 스크롤 진행도에 1:1 대응시킬 것.
   - 역방향 스크롤 시 자연스럽게 되감기 — scrub이 자동 처리.
   - 캔버스나 복잡한 방식 대신, 레이아웃 위에 떠 있는 **전역 오버레이 요소** 방식을 선호

## 10. 헤더 / 네비게이션

### 구조

- `position: fixed`, 투명 배경 — 별도 헤더 바 없음. 섹션 배경 위에 float
- 로고: 왼쪽 상단 / 햄버거: 오른쪽 상단
- 히어로에서는 헤더가 히어로 콘텐츠의 일부 — 빅타이포가 히어로 안에 있다가 스크롤하면서 헤더 자리로 이동. 히어로와 헤더가 분리된 게 아니라 하나로 연결된 구조.

### 로고 시스템

**브랜드 어원**: Rectangle에서 시작. rec**t**angle의 t를 + 로 표현 → REC+ANGLE = 네모결

**로고 컬러**

| 요소      | 상태          | 표현               |
| --------- | ------------- | ------------------ |
| 네모 / ON | 기본 텍스트값 | —                  |
| △         | 틸 filled     | `#0891b2` 채움     |
| ○         | 틸 outlined   | `#0891b2` 테두리만 |

### 컬러

| 상태           | 로고/햄버거 색 |
| -------------- | -------------- |
| 배경 어두울 때 | `#ffffff`      |
| 배경 밝을 때   | `#0d1a1f`      |

> 배경색 인터폴레이션과 연동하여 자동 전환

### 햄버거 버튼

- 네모 테두리 안에 세 줄 (`≡`)
- 배경: 투명 + 테두리만 (색은 로고와 동일하게 연동)
- 닫힘: `≡` / 열림: `×` (선이 rotate되며 변형)
- OFF 상태: 숨김 / ON 전환 완료: fade-in

### 사이드 메뉴

- 메인 콘텐츠가 왼쪽으로 밀리면서 뒤에 있던 메뉴가 드러나는 방식 (참조: outspring.co.kr)
- 메뉴 배경: `#0891b2` (브랜드 틸)
- 텍스트: `#ffffff` (흰색)
- 호버: `#E8734A` (오렌지) — 개발하면서 확정
- 메뉴 항목: 크게, 여백 넉넉하게 (에디토리얼 그리드 언어)

참고 https://www.outspring.co.kr/

### Logo Transform (홈 전용)

히어로부터 ForWho까지 스크롤하면서 로고가 브랜드 어원을 경험하게 하는 여정.
OFF→ON 전환은 딱 한 번에 교체, 나머지는 스크롤 progress에 따라 점진적으로 변형.
Fluid Typography (`clamp()`) 적용 — 뷰포트에 따라 빅타이포 크기 자동 조절.

```
히어로 OFF (정지)
└── 빅타이포: 네모△/○OFF (헤더 자리, 상단)
    햄버거: 숨김

OFF → ON 전환 (트리거 방식 미정)
└── 네모△/○OFF → 네모△/○ON (딱 한번에)
    햄버거: fade-in 등장

히어로 → Pain & 공명 (스크롤 — Logo Transform)
└── △/○ON 빠짐 → 네모만 남음

#### 구현 방식

> **"브랜드의 어원을 경험하게 하는 8단계 'Logo Journey' 인터랙션으로 구현. 헤더 로고가 단독 요소로 존재하며 스크롤에 따라 상태가 변하는 방식."**
>
> **1. 기본 원칙**:
>
> - **Single Source**: 헤더 로고와 히어로 빅타이포를 별도로 만들지 말고, 하나의 
>   Header 컴포넌트 내 `div`가 모든 섹션을 관통하며 유지될 것.
> - **Fluid Typography**: `clamp()`를 사용하여 뷰포트에 따라 로고 크기가 자동 조절되게 할 것.
> - **Scroll Sync**: 모든 변형은 GSAP ScrollTrigger의 `scrub`을 통해 스크롤 진행도에 1:1로 대응할 것.
>
> **2. 로고 구성 규칙**:
>
> - 우리 브랜드명은 **한글 '[네모]' + 도형 '[△/○]' + 영문 '[ON]'**의 조합이다.
> - '[△/○]'는 콜론(:) 역할을 하는 **세모와 동그라미 도형**이다.
> - 영문 'NEMO'는 절대로 사용하지 않으며, 오직 한글 '네모'만 사용한다.
>
> **3. 8단계 여정 설계**:
>
> - (1) **Hero OFF**: 헤더 위치에서 빅타이포 크기의 **[네모] + [△/○] + [OFF]**로 시작.
> - (2) **OFF → ON**: 스크램블 효과와 함께 **[네모] + [△/○] + [ON]**으로 전환. 햄버거 등장.(이때만 트리거 방식)
> - (3) **Hero → Pain**: 도형 **[△/○]**과 **[ON]**은 소멸.
>   - **배경 트랜지션**: 프레이즈 [결] 뒤의 네모 박스(본문 요소)가 전체 화면으로 확장되며 배경(`#0D1A1F`)이 됨.
>   - **로고 트랜스폼**: 헤더 로고 텍스트 [네모](헤더 요소)는 빅타이포 상태에서 헤더 기본 크기로 작아지며 안착.
> - (4) **Pain 구간**: 헤더 위치에 '네모' 고정. 새 테두리 네모 등장.
> - (5) **Pain → Message**: '네모'에서 [RECTANGLE]로 텍스트 전환. 스크롤하며 'T' 문자가 '+' 기호로 모핑(Morphing).
> - (6) **Message 구간**: 'REC+ANGLE' 고정.
> - (7) **Message → ForWho**: **[REC+ANGLE]**이 다시 한글 **[네모]**로 돌아오고, 사라졌던 **[△/○]**와 **[ON]**이 다시 합쳐지며 완성.
> - (8) **마지막**: 최종 완성된 **[네모] + [△/○] + [ON]** 형태 유지.
>
> **4. 기술적 제약**:
>
> - 브랜드명은 무조건 한글 **'네모'**만 사용 (영문 NEMO 금지).
> - 전환 시 레이아웃이 튀지 않도록 `fixed` 위치와 `x, y` 트랜스폼을 조합할 것.

### 다른 페이지

- 헤더: `네모△/○ON` 고정
- Logo Transform 없음

**OFF 상태**

### 섹션 02. Pain & 공명

#### 배경색 / 색상 참조

`#0D1A1F` (섹션 다크 배경 토큰)
네모: `#f0ebe3` (크림 텍스트/요소 토큰)

#### 레이아웃 방식

풀블리드 다크 배경. 네모 약간 오른쪽 위치. 페인포인트 및 공명문장은 네모 안에 나옴

#### 반응형 레이아웃

네모 크기/위치 수치 조정 외 레이아웃 구조 변경 없음. 개발시 확정.

#### 스크롤 정책

- PC / 모바일 모두 세로 스크롤
- GSAP ScrollTrigger pin — 스크롤 progress에 모든 인터랙션 연동
- **끝까지 봐야 다음 섹션(메시지)으로 이동**
- 위로 스크롤 시 역순 되감기 → Pain 첫 번째까지 돌아와야 히어로로 이동
- 네모도 스크롤 progress에 따라 역방향으로 자연스럽게 움직임

#### 진행 시퀀스

**1단계 — Pain 포인트 5개**
스크롤마다 Pain 포인트 교체. 뷰포트 상단에 감정키워드 누적 (뒤로 스크롤 시 pop).

**2단계 — 네모 안에서 문장 등장**

스크롤에 의해 페인포인트 5개가 다 보이고 나면 스크롤마다 한 단락씩 등장.

```
나는,
당신은,
우리는 지금 설명 가능한가요?
```

**3단계 — FallingText 트리거**
스크롤 특정 지점 도달 시 (나는 당신은 우리는 지금 설명가능한가요?까지 다 나온 시점)
스크롤을 한번 더 하면 상단 누적 키워드들이 우수수 떨어짐

**4단계 — 네모 전환**
오른쪽 → 중앙 이동 + 테두리 → `#f0ebe3` 채워짐 (공명 문장이 나오기 전)

**5단계 — 공명 문장**

```
의미를 부여하면 의미가 됩니다.
의미가 있으면 선명해집니다.
선명해지면 설명이 가능해지고,
설명이 가능해지면 연결됩니다.
```

크림 박스 안, 텍스트 `#0D1A1F`

### 섹션 03. 메시지

#### 레이아웃 방식

**참고**: https://www.easymedia.net/kr/main (네모 배경 고정 + 텍스트 reveal 방식)

사이드 FUNNEL 레이어(영문/한글) — 모바일에서는 사이드 고정 대신 네모 박스 안 배경 텍스트로 처리.
FUNNEL(영문) + 브랜딩→디자인시스템→로고·웹/앱→자동화(한국어) 세로 나열, 흐릿하게 표시.
메시지 카피는 그 위 레이어로 올라옴. 네모 세로 박스 중앙 고정 구조는 유지.

#### 스크롤 정책

GSAP ScrollTrigger scrub. 스크롤 progress 연동 — 그룹 전체가 함께 올라오고 reveal되는 속도를 유저가 컨트롤.

#### 진행 시퀀스

**텍스트 올라오는 방식**

- 스크롤 progress 연동으로 그룹 전체가 함께 올라옴

**reveal 방식**

- 그룹 전체 올라온 후 stagger reveal (흐릿 → 선명)
- 첫 문장 reveal 진행 중 → 다음 문장 reveal 시작 (완전 동시 아님)
- reveal 완료 후 사라지고 다음 그룹 시작

**텍스트 컬러 시스템**

| 상태             | 배경 위   | 네모 위                 | 네모 안 FUNNEL 텍스트 위(모바일) |
| ---------------- | --------- | ----------------------- | -------------------------------- |
| Reveal 전 (흐릿) | `#C4BDB4` | `rgba(240,235,227,0.3)` | 개발시 확정                      |
| Reveal 후 (선명) | `#0D1A1F` | `#f0ebe3`               | 개발시 확정                      |

단순히 예쁜 웹사이트를 만들지 않습니다.

**③ 그룹** (scroll progress 연동 → 올라온 후 stagger reveal)

```
당신의 언어를 고객의 언어로.

당신의 감각을 고객이 이해하는 구조로.

#### 반응형 레이아웃

인디케이터 없음, 스와이프 힌트 모션으로 대체 (얇은 선 좌우 확장 애니메이션) — PC/모바일 동일.

```

> 브랜드 진단을 시작하시겠습니까?

[ YES ]  [ NO ]
```

**YES 선택 시**

```
> initializing brand_diagnosis...
> loading user_profile...
> connecting to NEMO:ON...
> ████████████████ 100%
> COMPLETE ✓
→ 브랜드 진단 페이지 자동 이동
```

**NO 선택 시**

```
> ████████████████ (중간에 멈춤)
> ERROR 404: brand_clarity not found
> 브랜드 진단이 필요합니다.
> redirecting...
→ 브랜드 진단 페이지 자동 이동
```
