# 헤더 로고 Transform 구현 — Cursor 프롬프트

## 개요

헤더 로고 Transform은 단순한 크기 변화가 아니다.
네모:ON이라는 브랜드 이름이 어디서 왔는지,
그 어원을 스크롤하면서 직접 경험하게 만드는 인터랙션이다.

히어로부터 ForWho까지 스크롤하는 동안
헤더 로고는 8단계의 여정을 거치며
브랜드의 탄생 과정을 시각적으로 보여준다.

OFF → ON → 네모 → RECTANGLE → REC+ANGLE → 네모△/○ON

이것이 이 인터랙션의 본질이다.

---

## 기술 스택

- Next.js App Router + TypeScript
- GSAP + ScrollTrigger (스크롤 연동 애니메이션)
- Lenis (스무스 스크롤) — GSAP ScrollTrigger와 연동
- Tailwind CSS
- Fluid Typography — `clamp()`로 뷰포트에 따라 빅타이포 크기 자동 조절

OFF → ON 전환은 트리거 방식(한 번에 교체).
나머지 모든 변형은 스크롤 progress에 1:1로 대응하는 scrub 방식.
역방향 스크롤 시 자연스럽게 되감기가 되어야 한다.

---

## 로고 구성 규칙

브랜드명은 **[네모] + [△/○] + [ON/OFF]** 의 조합이다.

- **[네모]** — 한글. 영문 NEMO는 절대 사용하지 않는다.
- **[△/○]** — 세모와 동그라미 도형. 콜론(:) 역할을 한다.
- **[ON/OFF]** — 영문.

---

## 8단계 여정

### 1단계 — Hero OFF (페이지 로드)

헤더 위치(왼쪽 상단)에 빅타이포 크기로 존재.

```
[네모] + [△/○] + [OFF]
```

햄버거 메뉴는 숨김 상태.
브랜드는 아직 켜지지 않았지만 이미 자리를 차지하고 있다.

---

### 2단계 — OFF → ON 전환 (트리거)

스크롤이 아닌 히어로 인터랙션 시퀀스에 의해 발생.
스크램블 효과와 함께 한 번에 교체된다.

```
[네모] + [△/○] + [OFF]  →  [네모] + [△/○] + [ON]
```

햄버거 메뉴 fade-in 등장.

**이 순간에 느껴져야 하는 경험**
브랜드가 켜지는 순간. 딱 한 번. 되감기 없다.

---

### 3단계 — Hero → Pain & 공명 (스크롤)

스크롤이 시작되면 [△/○]와 [ON]이 사라지고
[네모]만 남아 서서히 작아지며 헤더 로고 크기로 안착한다.

```
[네모] + [△/○] + [ON]  →  [네모] (헤더 크기로 축소)
```

위치는 왼쪽 상단 그대로.
threshold를 넘으면 position: fixed 전환.

**이 구간에서 느껴져야 하는 경험**
화려함이 사라지고 본질만 남는다.
[네모]라는 이름의 뿌리가 드러나기 시작한다.

---

### 4단계 — Pain & 공명 구간 (고정)

헤더 위치에 [네모] 고정.

```
[네모]
```

---

### 5단계 — Pain & 공명 → 메시지 (스크롤)

[네모]에서 [RECTANGLE]로 텍스트가 전환된다.
스크롤하면서 'T' 문자가 '+' 기호로 모핑된다.

```
[네모]  →  [RECTANGLE]  →  [REC+ANGLE]
```

**이 구간에서 느껴져야 하는 경험**
한글 네모의 어원이 드러난다.
Rectangle — 사각형. 그리고 T가 +로 변하며 의미가 해체되고 재구성된다.
보는 사람이 "아, 이래서 네모:ON이구나"를 느끼는 순간.

---

### 6단계 — 메시지 구간 (고정)

헤더 위치에 [REC+ANGLE] 고정.

```
[REC+ANGLE]
```

---

### 7단계 — 메시지 → ForWho (스크롤)

[REC+ANGLE]이 다시 한글 [네모]로 돌아오고
사라졌던 [△/○]와 [ON]이 합쳐지며 완성된다.

```
[REC+ANGLE]  →  [네모] + [△/○] + [ON]
```

**이 구간에서 느껴져야 하는 경험**
어원의 여정을 마치고 브랜드가 완성된다.
처음의 [네모△/○ON]으로 돌아왔지만
이제는 그 이름이 왜 이 이름인지 알고 있다.

---

### 8단계 — ForWho 이후 (고정 유지)

최종 완성된 형태로 고정.

```
[네모] + [△/○] + [ON]
```

이후 모든 섹션에서 이 상태로 유지된다.

---

## 구현 시 반드시 지켜야 하는 것

1. **Single Source 원칙**
   헤더 로고와 히어로 빅타이포는 하나의 DOM 요소다.
   별도로 만들지 않는다.

2. **한글 '네모' 고정**
   영문 NEMO는 절대 사용하지 않는다.

3. **OFF → ON은 트리거, 나머지는 scrub**
   2단계 전환만 트리거 방식(한 번에). 나머지는 스크롤 1:1 대응.

4. **역방향 되감기**
   3단계 이후는 위로 스크롤하면 되감긴다.
   단, 2단계(OFF→ON)는 되감기 없다.

5. **레이아웃 안정성**
   전환 시 레이아웃이 튀지 않도록 fixed 위치와 x, y transform을 조합한다.

6. **네모 모핑과의 관계**
   Logo Transform과 네모 모핑은 동시에 진행되는 구간이 있다.
   같은 레이어 구조 안에서 성능 충돌 없이 작동해야 한다.

---

## 구현 방식에 대하여

위에 서술한 것은 경험과 흐름이지 구현 방법이 아니다.
이 8단계 여정을 가장 잘 구현할 수 있는 방법을 판단해서 제안해줘.

전체 아키텍처를 먼저 제안해준 뒤 구현을 시작해줘.
네모 모핑 문서와 함께 참고해서
두 인터랙션이 하나의 아키텍처 안에서 충돌 없이 설계되도록 해줘.
