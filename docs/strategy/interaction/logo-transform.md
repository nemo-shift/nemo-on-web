# 🔡 Logo Journey & Transformation Guide (V11.34 Sync)

> **[V11.34 Implementation Note]**: 본 문서에 기술된 모든 로고의 상태(가시성, 모핑, 크기)는 `src/data/home/journey.ts` 마스터 시트를 통해 데이터로 제어됩니다. 수치 조정이 필요할 경우 로직 코드가 아닌 데이터 시트를 수정하십시오.

## 개요

헤더 로고 Transform은 단순한 크기 변화가 아니다.
네모:ON이라는 브랜드 이름이 어디서 왔는지,
그 어원을 스크롤하면서 직접 경험하게 만드는 인터랙션이다.

히어로부터 CTA까지 스크롤하는 동안
헤더 로고는 9단계의 여정을 거치며
브랜드의 탄생 과정을 시각적으로 보여준다.

OFF → ON → 네모 → RECTANGLE → REC+ANGLE → 네모△/○ON

이것이 이 인터랙션의 본질이다.

---

## 기술 스택

- Next.js App Router + TypeScript
- GSAP + ScrollTrigger (스크롤 연동 애니메이션)
- Lenis (스무스 스크롤) — GSAP ScrollTrigger와 연동
- Tailwind CSS
- Fluid Typography — `clamp()`로 뷰포트에 따라 빅타이포 크기 자동 조절

OFF → ON 전환은 트리거 방식(한 번에 교체).
나머지 모든 변형은 스크롤 progress에 1:1로 대응하는 scrub 방식.
역방향 스크롤 시 자연스럽게 되감기가 되어야 한다.

---

## 로고 구성 규칙

브랜드명은 **[네모] + [△/○] + [ON/OFF]** 의 조합이다.

- **[네모]** — 한글. 영문 NEMO는 절대 사용하지 않는다.
- **[△/○]** — 세모와 동그라미 도형. 콜론(:) 역할을 한다.
- **[ON/OFF]** — 영문.

---

## 9단계 여정

### 1단계 — Hero OFF (페이지 로드)

헤더 위치(왼쪽 상단)에 빅타이포 크기로 존재.

```
[네모] + [△/○] + [OFF]
```

햄버거 메뉴는 숨김 상태.
브랜드는 아직 켜지지 않았지만 이미 자리를 차지하고 있다.

---

### 2단계 — OFF → ON 전환 (트리거)

스크롤이 아닌 히어로 인터랙션 시퀀스에 의해 발생.
스크램블 효과와 함께 한 번에 교체된다.

```
[네모] + [△/○] + [OFF]  →  [네모] + [△/○] + [ON]
```

햄버거 메뉴 fade-in 등장.

**이 순간에 느껴져야 하는 경험**
브랜드가 켜지는 순간. 딱 한 번. 되감기 없다.

---

### 3단계 — Hero → Pain (스크롤)

스크롤이 시작되면 [△/○]와 [ON]이 사라지고
[네모]만 남아 서서히 작아지며 헤더 로고 크기로 안착한다.

```
[네모] + [△/○] + [ON]  →  [네모] (헤더 크기로 축소)
```

위치는 왼쪽 상단 그대로.
`logoMode: 'nemo-only'`

**이 구간에서 느껴져야 하는 경험**
화려함이 사라지고 본질만 남는다.
[네모]라는 이름의 뿌리가 드러나기 시작한다.

---

### 4단계 — Pain 구간 (고정)

헤더 위치에 [네모] 고정.

```
[네모]
```

공명(RESONANCE) 구간까지 유지.

---

### 5단계 — Pain → 메시지 (스크롤)

[네모]에서 [RECTANGLE]로 텍스트가 전환된다.

```
[네모]  →  [RECTANGLE]
```

`logoMode: 'rectangle'`

**이 구간에서 느껴져야 하는 경험**
한글 네모의 어원이 드러난다.
Rectangle — 사각형.

---

### 6단계 — 메시지 구간 (고정)

헤더 위치에 [RECTANGLE] 고정.

```
[RECTANGLE]
```

---

### 7단계 — 메시지 → ForWho (스크롤)

[RECTANGLE]에서 T가 +로 모핑되며 [REC+ANGLE]로 변형된다.

```
[RECTANGLE]  →  [REC+ANGLE]
```

`logoMode: 'plus'`

**이 구간에서 느껴져야 하는 경험**
T가 +로 변하며 의미가 해체되고 재구성된다.
보는 사람이 "아, 이래서 네모:ON이구나"를 느끼는 순간.

---

### 8단계 — ForWho → BrandStory (스크롤)

[REC+ANGLE]이 다시 한글 [네모]로 돌아오고
사라졌던 [△/○]와 [ON]이 합쳐지며 완성된다.

```
[REC+ANGLE]  →  [네모] + [△/○] + [ON]
```

`logoMode: 'nemo-on'`

**이 구간에서 느껴져야 하는 경험**
어원의 여정을 마치고 브랜드가 완성된다.
처음의 [네모△/○ON]으로 돌아왔지만
이제는 그 이름이 왜 이 이름인지 알고 있다.

---

### 9단계 — BrandStory → CTA (고정 유지)

최종 완성된 형태로 고정.

```
[네모] + [△/○] + [ON]
```

이후 모든 섹션에서 이 상태로 유지된다.

## 구현 시 반드시 지켜야 하는 것

1. **Single Source 원칙**
   헤더 로고(`JourneyLogo`)는 항상 `fixed` 레이어에 하나만 존재한다.
   히어로 빅타이포와 헤더 로고는 동일한 하나의 DOM 요소다.
   별도로 만들지 않는다.

2. **한글 '네모' 고정**
   영문 NEMO는 절대 사용하지 않는다.

3. **OFF → ON은 트리거, 나머지는 scrub**
   2단계 전환만 트리거 방식(한 번에). 나머지는 스크롤 1:1 대응.

4. **역방향 되감기**
   3단계 이후는 위로 스크롤하면 되감긴다.
   단, 2단계(OFF→ON)는 되감기 없다.

5. **레이아웃 안정성**
   전환 시 레이아웃이 튀지 않도록 fixed 위치와 scale transform을 조합한다.

6. **네모 모핑과의 관계**
   Logo Transform과 네모 모핑은 동시에 진행되는 구간이 있다.
   동일한 GSAP 마스터 타임라인의 라벨(L)을 공유하며 작동한다.

7. **데이터 시트 수정 원칙**
   수치 조정이 필요할 경우 `builders/hero.ts` 로직이 아닌
   `src/data/home/journey.ts` 마스터 시트를 수정한다.

## 구현 방식

### 폰트 기반 렌더링
로고는 폰트로 렌더링한다.
- **[네모]** — 이사만룬체 Light (로컬 woff2)
- **[ON/RECTANGLE 등]** — Gmarket Sans Medium (로컬 woff2)

### T → + 모핑
RECTANGLE의 'T' 글자가 '+' 기호로 변하는 모핑은
SVG 라인 3개(`JourneyLogo` 내부 `tLines`)로 처리한다.
실제 폰트 글자가 변형되는 게 아니라 라인이 움직이는 방식.

### Fixed Anchor 전략
`JourneyLogo`는 항상 최상위(`fixed`) 레이어에 존재한다.
히어로 섹션에서는 빅타이포 크기(`scale: BIG_SCALE`)로,
Pain 이후에는 헤더 크기(`scale: HEADER_SCALE`)로 GSAP이 직접 제어한다.
`HEADER_POS` 상수로 기기별 위치를 관리한다.

### 색상 제어
`--header-fg` CSS 변수를 GSAP `fromTo`로 구간별로 전환한다.
`builders/hero.ts`의 `lastEnv` 데이터 체이닝으로 무결성을 확보한다.
`invalidateOnRefresh` 옵션은 절대 추가하지 않는다 — 색상이 `rgba(0,0,0,0)`으로 박히는 버그가 재발한다.
