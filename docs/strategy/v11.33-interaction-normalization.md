# [V11.33] 인터랙션 정규화 및 아키텍처 정화 (Interaction Normalization) - 2026-04-04 구현

이 문서는 네모:ON의 인터랙션 위계를 4단계로 정밀화하고, 프로젝트 전반의 반응형 브레이크포인트를 '744px(tablet-p)'로 단일화하여 아키텍처적 무결성을 확보한 **'v11.33 정규화 엔진'**의 상세 기록을 담고 있습니다.

---

## 🏛️ 1. 인터랙션 마스터 (Interaction Mastery)

### [MenuToggle] 4단계 모핑 상태 머신
단순한 열기/닫기를 넘어, 사용자의 액션에 따른 즉각적이고 우아한 시각적 피드백을 위해 4단계 인터랙션 시퀀스를 확립했습니다.
- **[☰] Hamburger**: 기본 닫힘 상태.
- **[▷] ArrowRight**: 닫힌 상태에서 호버 시 '진입 의사' 표현.
- **[X] CloseX**: 열림 상태.
- **[◁] ArrowLeft**: 열린 상태에서 호버 시 '복귀 의사' 표현.
- **모바일 보정**: 터치 편의성을 위해 컨테이너 사이즈를 `36px`에서 **`40px`**로 상향 조정했습니다.

### [SideMenu] 4단계 반응형 너비 위계
뷰포트 해상도에 따라 사이드 메뉴가 차지하는 비중을 수학적으로 조율하여 모든 기기에서 최적의 여백을 유지합니다.
1.  **Mobile**: 100vw (전체 화면)
2.  **Tablet-Portrait (744px~)**: 75vw
3.  **Tablet-Landscape (992px~)**: 55vw
4.  **PC (1280px~)**: 35vw (황금 비율)

## 🏗️ 2. 아키텍처 정규화 (Architecture Purity)

### 전역 브레이크포인트 SSOT (Single Source of Truth)
표준 Tailwind의 `md:(768px)`를 폐기하고, 네모:ON 전용 규격인 **`tablet-p:(744px)`**로 전역(23개 파일) 신경망을 일치시켰습니다.
- **배경**: 아이패드 미니 등 주요 모바일 태블릿의 세로 해상도(744px)를 기점으로 레이아웃이 동시에 기지개를 켜도록 설계.
- **설정 정화**: `tailwind.config.ts`에서 불필요한 기본값들을 제거하고 정규 5단계 위계로만 시스템을 구동.

### 데이터 무결성 가드 (Data Integrity Guard)
시스템이 타임라인을 그리는 '태초의 순간'부터 진실을 인지하도록 하드닝 작업을 수행했습니다.
- **`hero.ts`**: 초기화 시점(`lastEnv`)이 이제 **`isOn`** 상태를 선제적으로 인식합니다.
- **안전한 폴백**: 옵셔널 체이닝(`on?.env`)을 적용하여 데이터가 부재한 스테이지에서도 시스템이 붕괴되지 않고 기본 상태를 유지하도록 설계했습니다.

## 🛡️ 3. 시스템 복원 및 하드닝 (System Resilience)

### 인코딩 장애 극복 히스토리
PowerShell 일괄 치환 과정에서 발생한 UTF-8 파편화 현상을 감지하고, 이를 .NET 기반 강제 인코딩 수술을 통해 복구했습니다.
- **Restoration**: 훼손되었던 한글 메타데이터와 주석들을 수동으로 정교하게 복원하여 코드의 인문학적 가치를 회복했습니다.
- **전방위 딤(Dim) 처리**: 사이드 메뉴 확장 시 헤더뿐만 아니라 섹션 본문의 텍스트 영역까지 딤 효과가 적용되도록 타겟 영역을 확장했습니다.
