# 네모:ON 홈페이지 구현 마스터 프롬프트

## 이 문서의 역할

이 문서는 네모:ON 홈페이지 구현의 전체 구조와 순서를 정의한다.
개별 요소 프롬프트(네모 모핑, Logo Transform)보다 먼저 읽어야 한다.
콘텐츠 디테일보다 구조와 순서를 먼저 잡는다.

## 핵심 구조 이해

이 홈페이지는 여러 요소가 동시에 맞물리는 구조다.

| 요소                    | 역할                                           | 작동 구간                       |
| ----------------------- | ---------------------------------------------- | ------------------------------- |
| **Logo Transform**      | 헤더 로고가 8단계 브랜드 어원 여정을 거침      | 히어로 ~ ForWho                 |
| **네모 모핑**           | 하나의 네모 오브젝트가 변형되며 섹션을 관통    | 히어로 ~ ForWho                 |
| **배경색 인터폴레이션** | 섹션 전환 구간에서 배경색이 스크롤에 따라 변화 | Pain→Message, ForWho→BrandStory |
| **Lenis + GSAP**        | 위 세 요소 모두의 기반                         | 전체                            |

이 요소들은 모두 같은 스크롤 타임라인 위에서 동시에 움직인다.
아키텍처를 먼저 잡고 요소를 올리는 순서로 구현한다.

## 레이어 계층 구조 (Z-index)

```
LV 4 (Top) — 전역 헤더 & 햄버거 메뉴
LV 3       — JourneyLogo (Logo Transform 텍스트)
LV 2       — 네모 오브젝트 (Pain 섹션 테두리 네모 / Shared Element)
LV 1       — 배경 확장 네모 (히어로 [결] 뒤 박스 → Pain 배경으로 확장)
LV 0       — 섹션 콘텐츠
```

## 섹션 순서 및 스크롤 조건

```
히어로
├── 스크롤 잠금 — OFF→ON 전환 시퀀스 동안 스크롤 비활성
├── OFF 상태: 빅타이포 [네모△/○OFF], 배경 #0a0a0a
├── ON 전환: 트리거 방식 (스크롤 아님)
└── ON 완성 후: 스크롤 활성화

Pain & 공명
├── 스크롤 진행 — Logo Transform + 네모 모핑 동시 진행
├── 배경: #0D1A1F (히어로 네모가 확장되며 생성)
└── 배경색 고정 구간

메시지
├── 스크롤 진행 — Logo Transform + 네모 모핑 동시 진행
└── 배경색 인터폴레이션: #0D1A1F → #f7f1e9

ForWho
├── Logo Transform 완성 (네모△/○ON 고정)
├── 네모 모핑 완성 (버스 사진 가로 박스)
└── 배경: #f7f1e9

브랜드 스토리
└── 배경색 인터폴레이션: #f7f1e9 → #0891b2

CTA (브랜드 진단)
└── 배경: #0a0a0a (다크)

푸터
├── position: fixed 하단 고정
├── 메인 콘텐츠가 위로 올라가며 드러나는 방식
└── 배경: #0a0a0a
```

## 전역으로 먼저 잡아야 할 것들

섹션 디테일 들어가기 전에 전역 세팅을 먼저 완성한다.

### 1. 디자인 토큰 (CSS 변수)

```css
--brand: #0891b2 --accent: #e8734a --bg-dark: #0a0a0a --bg-section-dark: #0d1a1f
  --bg-cream: #f7f1e9 --text-light: #f0ebe3 --text-dark: #0d1a1f
  --bg: (스크롤 인터폴레이션용 동적 변수);
```

히어로 OFF/ON 전용 컬러는 별도 관리 (전역 토큰과 분리).

### 2. 폰트 시스템

| 분류          | 폰트                | 용도                   |
| ------------- | ------------------- | ---------------------- |
| 한글 본문     | SUIT Variable       | 전반 한글 (로컬 woff2) |
| 영문 본문     | DM Sans             | 영문 본문/UI           |
| 영문 빅타이포 | Bebas Neue          | 영문 헤딩              |
| 터미널        | DM Mono             | CTA 섹션 전용          |
| 로고 [네모]   | 이사만룬체 Light    | 로고 전용 (로컬 woff2) |
| 로고 [ON]     | Gmarket Sans Medium | 로고 전용 (로컬 woff2) |

Fluid Typography — `clamp()`로 뷰포트에 따라 자동 조절.

### 3. 데이터 구조

콘텐츠 데이터와 설정값 분리.

```
src/
├── constants/
│   ├── colors.ts     # 컬러 토큰
│   └── hero.ts       # 히어로 타이밍값
├── data/
│   ├── home/
│   │   ├── pain.ts        # Pain 포인트 5개
│   │   ├── forwho.ts      # ForWho 타겟 5개
│   │   └── message.ts     # 메시지 섹션 카피
│   └── index.ts
└── types/
```

### 4. 커스텀 커서 (PC 전용, 전역)

`pointer: fine` 환경에서만 활성.
기본: 작은 도트 + 지연 추적 링.
링크/버튼/타이틀 호버: 링이 사각형으로 변형.
푸터 Get in touch 호버: PointRingCursor 예외 — 흰 원형 "contact" 커서로 교체.

### 5. 사이드 메뉴 구조

열릴 때 헤더 + 콘텐츠 + 푸터 전체 래퍼가 함께 왼쪽으로 밀림.
푸터만 따로 움직이지 않음.

### 6. 푸터 구조

`position: fixed` 하단 고정.
메인 콘텐츠 전체가 스크롤하면 위로 올라가며 푸터가 드러나는 방식.
CSS만으로 구현 가능.

## 반응형 정책

### 레이아웃 breakpoint

| breakpoint | 범위           |
| ---------- | -------------- |
| PC         | ≥ 1024px       |
| 태블릿     | 768px ~ 1023px |
| 모바일     | < 768px        |

태블릿 가로(≥ 1024px)는 PC 레이아웃으로 처리.

### 인터랙션 분기

레이아웃과 인터랙션은 독립적으로 작동.

| 조건        | 판단 기준         | 적용                                   |
| ----------- | ----------------- | -------------------------------------- |
| 마우스 있음 | `pointer: fine`   | 커스텀 커서 활성, 호버 인터랙션 활성   |
| 터치만 있음 | `pointer: coarse` | 커스텀 커서 비활성, 터치 인터랙션 활성 |

### 반응형 구현 원칙

- 레이아웃 구조는 처음부터 반응형으로 잡는다.
- Logo Transform과 네모 모핑은 position: fixed 기반이라 뷰포트 크기에 따라 위치/크기 계산이 달라진다. 레이어 구조 잡을 때 반응형을 함께 고려한다.
- 인터랙션 복잡도는 PC 먼저 완성 후 모바일 간소화 버전으로 대응한다.
- 모바일에서 Logo Transform, 네모 모핑의 구체적인 간소화 방식은 개발자가 판단한다.

## 구현 순서

콘텐츠 디테일은 나중에. 구조가 먼저다.
각 Step의 확인 기준을 만족한 후 다음 Step으로 넘어간다.

### Step 1 — 스크롤 인프라

Lenis + GSAP ScrollTrigger 연동 세팅.
스크롤 활성/비활성 전환 가능한 구조 포함 (히어로 OFF→ON 동안 잠금 필요).

**확인 기준**: 스무스 스크롤 작동. ScrollTrigger가 Lenis scroll progress를 정확히 추적.

---

### Step 2 — 전역 세팅

CSS 변수(디자인 토큰), 폰트 로드, 데이터 구조, 커스텀 커서, 반응형 breakpoint.
빈 껍데기로 전역 구조 완성.

**확인 기준**: 폰트 정상 로드. CSS 변수 전역 참조 가능. breakpoint 분기 작동.

---

### Step 3 — 레이어 구조 확정

콘텐츠 없이 빈 박스로 레이어 계층 확정.
PC / 태블릿 / 모바일 모두에서 레이어가 올바르게 쌓이는지 확인.

**확인 기준**: 4개 레이어가 올바른 z-index로 쌓힘. fixed 요소들이 스크롤해도 제자리 유지.

---

### Step 4 — 푸터 구조

`position: fixed` 하단 고정. 콘텐츠가 위로 올라가며 드러나는 방식.

**확인 기준**: 스크롤 시 콘텐츠 뒤에서 푸터가 자연스럽게 드러남.

---

### Step 5 — 히어로 OFF 상태

빅타이포 [네모△/○OFF] 왼쪽 상단. 배경 #0a0a0a. 스크롤 잠금 상태.
반응형 레이아웃 함께 적용.

**확인 기준**: OFF 상태 렌더링 정확. 반응형 레이아웃 분기 작동.

---

### Step 6 — Logo Transform 뼈대

텍스트 변형 없이 크기 변화만 먼저.
OFF→ON 전환(트리거) → 헤더 크기 축소 → 역방향 되감기.

**확인 기준**: 스크롤에 따라 로고 부드럽게 축소. 역방향 되감기 작동.

---

### Step 7 — 네모 모핑 뼈대

색상, 이미지 없이 크기와 위치 변형만.
히어로 작은 네모 → Pain 배경 확장 → 테두리 네모 → 세로 박스 → 가로 박스.

**확인 기준**: 단계별 변형 작동. 역방향 되감기 작동.

---

### Step 8 — 두 요소 동시 작동 확인

Logo Transform + 네모 모핑이 같은 타임라인에서 충돌 없이 작동하는지.

**확인 기준**: 동시 작동 시 성능 저하 없음. 타이밍 충돌 없음.

---

### Step 9 — 배경색 인터폴레이션

GSAP ScrollTrigger onUpdate 콜백에서 lerp로 CSS 변수(`--bg`) 실시간 주입.

| 구간                   | 시작      | 끝        |
| ---------------------- | --------- | --------- |
| Pain & 공명 → 메시지   | `#0D1A1F` | `#f7f1e9` |
| ForWho → 브랜드 스토리 | `#f7f1e9` | `#0891b2` |

**확인 기준**: 전환 구간에서 배경색이 스크롤에 따라 부드럽게 변화.

---

### Step 10 — Logo Transform 텍스트 변형

8단계 여정 구현.
(상세: cursor-logo-transform-prompt.md 참고)

---

### Step 11 — 네모 모핑 색상 및 이미지

색상 전환, 테두리, 버스 사진.
(상세: cursor-nemo-morphing-prompt.md 참고)

---

### Step 12 — 섹션 콘텐츠

구조 완성 후 콘텐츠 올리기.
히어로 OFF→ON 시퀀스, Pain 포인트, 공명 문장, 메시지, ForWho 순서대로.

## 구현 제안 (유연하게 참고)

아래는 현재 가장 검증된 방식을 제안하는 것이다.
더 나은 방법이 있다면 그것을 선택한다.

**스크롤 인프라**
Lenis + GSAP ScrollTrigger 조합이 현재 가장 검증된 방식.
`lenis.on('scroll', ScrollTrigger.update)` 패턴으로 연동.

**Logo Transform + 네모 모핑 아키텍처**
두 요소를 하나의 GSAP Context 또는 타임라인으로 묶어 관리하면
타이밍 충돌을 줄이고 역방향 되감기를 일관되게 처리할 수 있다.

**배경색 인터폴레이션**
GSAP ScrollTrigger `onUpdate` 콜백에서 `gsap.utils.interpolate()`로
CSS 변수에 직접 주입하는 방식이 가볍고 안정적이다.

**반응형 + position: fixed 요소**
fixed 요소의 위치/크기 계산에 `vw`, `vh`, `clamp()`를 활용하면
breakpoint마다 별도 계산 없이 자연스럽게 대응 가능하다.

**푸터**
CSS만으로 구현 가능. 메인 래퍼에 `position: relative`, 푸터에 `position: fixed; bottom: 0; z-index: -1` 구조.
