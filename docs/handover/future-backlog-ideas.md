## 📋 향후 고도화 백로그 (Future Backlog)

특정 마일스톤(로고 최종 확정, 콘텐츠 주입 완료 등) 이후에 수행할 예정된 과제들입니다.

### ⚙️ [Technical Debt] 아키텍처 하드닝

- [ ] **[Step 3: 초정밀 미세 좌표 보간]**: 브라우저 상단 바 이동 등 미세 리사이즈 시 발생하는 1~10px의 좌표 오차를 수동 갱신(`refresh`) 및 이징으로 보정.
      **현상**: 모바일 브라우저 주소창이 나타나거나 사라질 때, 혹은 윈도우 크기 변경 시 화면이 아주 미세하게(1~10px) 덜컹거리는 지터(Jitter) 현상.
      **방안**: 눈에 보이지 않는 미세한 오차를 강제로 다시 계산(`ScrollTrigger.refresh()`)하고 GSAP 이징을 결합하여, 스크롤이 단 1픽셀도 튀지 않게 부드럽게 고정.
      **목표**: 어떤 디바이스 환경에서도 시각적 흔들림 없는 완벽한 스태빌라이징 구현.

- [ ] **[System Cleanup]**: 성능 확인 및 디버깅을 위해 심어둔 수많은 임시 주석 및 불필요한 코드 조각 전수 제거.
      **현상**: 개발 중 삽입된 대량의 메모성 주석과 테스트 로직들이 코드의 가독성을 저해하고 유지보수 시 노이즈가 됨.
      **방안**: 현재 'Active console.log'는 존재하지 않으나, 전역 스캔을 통해 주석 처리된 로그와 잔재를 모두 소탕하여 코드의 순수성능 극대화.

- [ ] **[Pure TS Expansion]**: `hero.ts`, `message.ts` 등 타 섹션 빌더들로 `{ current: boolean }` 순수 타입 아키텍처 확산 적용.
      **현상**: `pain.ts`를 제외한 일부 구형 빌더들이 런타임 데이터에 의존하고 있어 에러 발생 시 추적이 어려움.
      **방안**: `pain.ts`에서 검증된 '순수 타입 사수 방식'을 전체 빌더로 확산하여 데이터 시발점의 무결성 확보.
      **비전**: 전 섹션의 데이터를 예측 가능하게 관리하여 유지보수 효율 200% 상향.

### 🎨 [Creative Polish] 브랜드 연출 고도화

- [x] **[공명 시퀀스 시네마틱 정밀 튜닝 (PC & Mobile)]**: **성료 (V11.41)**
    - 터치 슬라이드 '1스크롤 1문장' 반응성 및 4단계 부상 애니메이션 완성.
    - PC 마퀴 절대 좌표(`vw`) 기반 오프스크린 무결성 확보.
    - 모바일 폰트 크기 최적화 및 태블릿 수직 위치 보정 완료.

- [ ] **[슬로건 엔트리 애니메이션]**: 이중 슬로건이 중앙의 'Switch' 축을 기점으로 좌우에서 미끄러지듯 등장하는 시네마틱 오프닝 연출.
      **현상**: 페이지 시동 시 슬로건들이 평면적으로 등장하여 브랜드의 시네마틱한 무드를 충분히 전달하지 못함.
      **방안**: 중앙 수직축을 기점으로 공중에서 자석처럼 달라붙는 슬라이드 인 및 페이드 연출 구현.

- [ ] **[다이내믹 타이포 상호작용]**: 토글 스위치 호버 시 슬로건의 투명도나 블러(Blur) 값이 실시간으로 반응하여 물리적 깊이감 강조.
      **현상**: 브랜드 핵심 인터랙션인 '토글 호버' 상황에서 시각적 깊이감과 입체적인 반응이 부족함.
      **방안**: 호버 시 배경 슬로건의 블러와 투명도를 실시간으로 조절하여 '공간감'이 살아있는 인터랙션 완성.

- [ ] **[브랜드 프리로더 & 시각적 햅틱]**: 사이트 진입 서사 및 키워드 투하 시의 물리적 실재감 부여.
      **현상**: 초기 로딩 공백의 적막함과 키워드 충돌 시의 시각적 무게감 결여.
      **방안**: 서사적 프리로더 연출과 키워드 투하 시 바닥 충격에 따른 미세한 화면 진동(Shake)을 추가하여 객체의 물리적 실재감 극대화.

### 🛡️ [UX & Optimization] 안정성 및 편의성

- [ ] **[Hitbox Sync]**: 로고 클릭 영역(Hitbox) 시각적 동기화. (로고가 축소됨에 따라 클릭 범위도 물리적으로 자동 리사이징) >>>>>>> 꼭 해야할 것
      **현상**: 로고가 축소된 후에도 클릭 가능 영역이 초기 빅 타이포 크기로 유지되는 현상.
      **방안**: 히어로 섹션 이탈 시 클릭 히트박스를 물리적으로 축소하거나, 로고 내부 콘텐츠로 이벤트를 이전.
      **방침**: 섹션별 연출 디테일이 100% 확정된 '최종 폴리싱' 단계에서 집도하며,
      **구현 시점에 지금의 제안보다 더 최적의(유연한/안정적인) 대안이 있다면 기꺼이 채택하는 유연성을 유지함.**

- [ ] **[Edge-Case Optimization]**: iPad Pro 등 특정 고사양 태블릿 및 저사양 기기(Graceful Degradation) 대응 최적화.
      **현상**: 특수한 화면 비율(iPad)이나 하드웨어 성능이 극도로 낮은 기기에서의 퍼포먼스 미검증.
      **방안**: 기기별 환경 가드를 정밀화하여 극한의 상황에서도 컨텐츠 가독성을 보존하고, 저사양 기기에서는 애니메이션 일부를 자동 간소화하는 방어적 UX 설계.

- [ ] **[Pointer Mode Transition]**: Touch/Mouse 모드 간의 실시간 전환 테스트 및 반응성 폴리싱.
      **현상**: 모바일 환경에서 외장 마우스 연결 등 입력 장치 변화 시 `interactionMode`의 실시간 반응성 미흡.
      **방안**: 입력 장치 변화를 실시간 감지하여 그에 맞는 UI 동작(호버 효과 활성화 등)으로 즉각 전환.
